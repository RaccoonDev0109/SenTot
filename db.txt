## Database Spec (PostgreSQL) – SenTot Backend for Website (As-Implemented)

Tài liệu này mô tả **database thực tế đang được implement** theo `prisma/schema.prisma` trong project backend.

---

## Quy ước chung (thực tế trong code)

- **Primary key**: `id` dạng **UUID** (Prisma `@default(uuid())`). Khi test API, các `:id` đều validate UUID (v1-v5).
- **Timestamps**:
  - đa số bảng có `created_at`, `updated_at` (timestamptz)
  - một số bảng chỉ có `created_at`
- **Quan hệ**: khai báo bằng FK trong Prisma; nhiều nơi có `onDelete: Cascade/SetNull`.
- **Enum**: dùng enum Prisma (tương đương enum trong DB khi migrate).

---

## 0) Enums

- `UserStatus`: `active | locked`
- `PetSpecies`: `dog | cat | bird | rabbit | other`
- `PetGender`: `male | female`
- `VaccinationStatus`: `completed | upcoming | overdue`
- `ClinicStatus`: `draft | active | suspended`
- `AppointmentStatus`: `pending | confirmed | completed | cancelled | no_show`
- `ReviewStatus`: `visible | hidden`
- `SupportTicketStatus`: `open | in_progress | resolved | closed`

---

## 1) Identity & Access

### 1.1 `users`

- `id` (uuid, PK)
- `full_name` (text/varchar, not null)
- `email` (unique, not null)
- `phone` (unique, not null)
- `avatar_url` (text, null)
- `password_hash` (text, not null)
- `status` (`UserStatus`, default `active`)
- `created_at` (timestamptz, default now)
- `updated_at` (timestamptz, auto update)

### 1.2 `roles`

- `id` (uuid, PK)
- `code` (unique, not null) — `customer | clinic_staff | admin`
- `name` (not null)
- `created_at`, `updated_at`

### 1.3 `user_roles`

- `user_id` (FK → users.id)
- `role_id` (FK → roles.id)
- **PK** (`user_id`, `role_id`)

### 1.4 `refresh_tokens` (JWT refresh rotation)

- `id` (uuid, PK) — dùng như `jti`
- `user_id` (FK → users.id)
- `token_hash` (sha256(refreshToken))
- `expires_at` (timestamptz)
- `revoked_at` (timestamptz, null nếu còn hiệu lực)
- `created_at`

Index:
- index(`user_id`)

### 1.5 `password_reset_tokens` (chưa expose API, nhưng đã có bảng)

- `id` (uuid, PK)
- `user_id` (FK → users.id)
- `token_hash`
- `expires_at`
- `used_at` (nullable)
- `created_at`

## 2) Pets

### 2.1 `pets`

- `id` (uuid, PK)
- `owner_id` (FK → users.id)
- `name` (not null)
- `species` (`PetSpecies`)
- `breed` (not null)
- `gender` (`PetGender`)
- `birth_date` (date)
- `weight_kg` (decimal(5,2))
- `avatar_key` (nullable) — giai đoạn 1 map ảnh sẵn
- `description` (nullable)
- `created_at`, `updated_at`

Index:
- index(`owner_id`)

### 2.2 `pet_weight_records` (đã có bảng, chưa expose API)

- `id` (uuid, PK)
- `pet_id` (FK → pets.id)
- `weight_kg` (decimal(5,2))
- `measured_at` (timestamptz)
- `note` (nullable)
- `created_at`

Index:
- index(`pet_id`, `measured_at`)

## 4) Clinics (Directory)

### 4.1 `clinics`

- `id` (uuid, PK)
- `name` (not null)
- `address` (not null)
- `phone` (not null)
- `description` (not null)
- `rating` (decimal(2,1), default 0)
- `review_count` (int, default 0)
- `open_hours_text` (not null)
- `cover_image_url` (nullable)
- `latitude` (decimal(9,6), nullable)
- `longitude` (decimal(9,6), nullable)
- `status` (`ClinicStatus`, default `active`)
- `created_at`, `updated_at`

Index:
- index(`name`)

### 4.2 `clinic_services`

- `id` (uuid, PK)
- `clinic_id` (FK → clinics.id)
- `name` (not null)
- `created_at`

Constraints:
- unique(`clinic_id`, `name`)

### 4.3 `clinic_facilities` (đã có bảng, chưa expose API)

- `id` (uuid, PK)
- `clinic_id` (FK)
- `name`
- `created_at`

### 4.4 `clinic_gallery_images` (đã có bảng, chưa expose API)

- `id` (uuid, PK)
- `clinic_id` (FK)
- `image_url`
- `sort_order` (default 0)
- `created_at`

### 4.5 `clinic_working_hours`

- `id` (uuid, PK)
- `clinic_id` (FK)
- `day_of_week` (0..6)
- `open_time` (time, nullable)
- `close_time` (time, nullable)
- `is_closed` (bool, default false)
- `created_at`, `updated_at`

### 4.6 `clinic_staff`

- `clinic_id` (FK)
- `user_id` (FK)
- `title` (nullable)
- **PK** (`clinic_id`, `user_id`)

## 5) Appointments

### 5.1 `appointments`

- `id` (uuid, PK)
- `pet_id` (FK → pets.id)
- `clinic_id` (FK → clinics.id)
- `date_time` (timestamptz)
